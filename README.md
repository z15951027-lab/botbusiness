# Chat Manager

✨ Chat Manager - помощник для Telegram Business. Он работает в личных Business-чатах: может включать автоответ "я занят", предупреждать за мат, удалять спам, мутить отдельные чаты и чистить последние сообщения.

## Быстрый смысл

- Один бот может обслуживать нескольких людей.
- Доступ выдается одноразовым кодом.
- Каждый клиент сам вводит имя, и бот подставляет его в автоответы.
- Настройки "я занят везде" и "анти-мат везде" отдельные для каждого клиента.
- Команды в Business-чате работают только для владельца этого Business-подключения.

## Доступ клиента: код или звёзды

Теперь у клиента на `/start` есть две кнопки:

- **У меня есть код приглашения** — обычный сценарий ниже.
- **Купить ключ за 25 ⭐** — бот выставляет счёт в Telegram Stars (валюта `XTR`, провайдер не нужен). После оплаты бот сразу просит имя для автоответов, как будто клиент ввёл код.

Оплата звёздами доступна всем, кроме главного админа (`ADMIN_USER_ID`) — для него открывается отдельная админ-панель, а не окно оплаты/кода.

## Как дать доступ клиенту

Главный админ из `ADMIN_USER_ID` пишет боту в личку:

```text
/invite
```

Бот выдаст одноразовый код, например:

```text
ABCD-1234-WXYZ
```

Отправь клиенту короткую инструкцию:

```text
1. Откройте бота и нажмите /start.
2. Отправьте код приглашения.
3. Напишите имя для автоответов.
4. Подключите бота в Telegram Business.
5. Откройте /settings и включите нужные функции.
```

Код работает только один раз. Для нового клиента создай новый код через `/invite`.

## Админ-панель: кодовое слово

Команды `/invite`, `/users`, `/deluser` больше не показаны в меню бота и не работают сразу — даже для главного админа. Чтобы их разблокировать:

1. Напиши боту в личку кодовое слово: `Админ`.
2. Бот подтвердит разблокировку и покажет список доступных команд.
3. После этого `/invite`, `/users`, `/deluser` и кнопка «Клиенты» в `/settings` работают как обычно.

Разблокировка проверяется по `ADMIN_USER_ID` — если кодовое слово напишет кто-то другой, ничего не произойдёт. Разблокировка сохраняется в базе, пока её явно не сбросить (удалив соответствующую запись из таблицы `settings`).

## Как смотреть и удалять клиентов

Список клиентов:

```text
/users
```

Бот откроет inline-меню. Каждый клиент будет отдельной кнопкой. По нажатию откроется карточка: имя, Telegram ID, username, статистика, режимы и кнопка удаления.

Удалить клиента:

```text
/deluser 123456789
```

Или открой `/users`, нажми на клиента и выбери `Удалить клиента`. Через кнопки удаление идет с подтверждением.

У главного админа такая же кнопка `Клиенты` есть в `/settings`.

После удаления бот отключит Business-подключения клиента и удалит его настройки. Команды `/users` и `/deluser` доступны только главному админу из `ADMIN_USER_ID`.

## Что делает клиент после кода

1. Пишет боту `/start`.
2. Отправляет одноразовый код.
3. Вводит имя, например `Артем`.
4. Подключает бота в Telegram Business.
5. Дает права на чтение, ответы и удаление сообщений.
6. Открывает `/settings`.

После этого автоответ будет выглядеть так:

```text
Артем сейчас занят и ответит позже.
Можете оставить сообщение, я передам его в отдел ожидания.
```

## Главная панель

В обычной личке с ботом:

```text
/settings
```

В панели есть:

- `Анти-мат во всех чатах` - включает предупреждения за мат во всех подключенных чатах.
- `Автоответ 'я занят'` - включает автоответ для всех чатов.
- `Статистика` - показывает статистику именно этого владельца.
- `Что умеет` - показывает команды.

## Тест отдельных сообщений

В обычной личке с ботом можно проверить отправку нескольких отдельных сообщений:

```text
/spamtest 10 текст
```

Бот отправит до 20 отдельных сообщений только в личку с тобой. В Business-чаты эта команда не работает.

## Команды в Business-чате

Эти команды пишутся прямо в переписке с человеком, где подключен Telegram Business:

```text
.status
```

Показать статус текущего чата. Сообщение статуса удалится через 15 секунд.

```text
.busy
```

Включить или выключить автоответ "я занят" только в этом чате.

```text
.skip
```

Исключить этот чат из общего режима "я занят". Повторная команда вернет чат обратно.

```text
.mat
```

Включить или выключить анти-мат только в этом чате.

```text
.mute
```

Молча удалять новые сообщения собеседника в этом чате навсегда.

```text
.mute 10m
```

Включить мут на время. Поддерживаются `s`, `m`, `h`, `d`: например `.mute 30s`, `.mute 10m`, `.mute 2h`, `.mute 1d`.

```text
.unmute
```

Снять мут.

```text
.clean 50
```

Удалить последние 50 сообщений, которые бот видел в этом чате. Если сообщения удалили вручную, Telegram сообщает об этом боту, и бот убирает их из своей базы. Поэтому следующие `.clean 10` берут следующие реальные сообщения.

```text
.on
```

Включить тестовый режим для текущего Business-чата.

```text
.off
```

Выключить тестовый режим.

```text
.spam 10 текст
```

Тестовая команда для чата с включенным `.on`: отправляет до 30 отдельных сообщений с задержкой 0.5 секунды. В обычных чатах не работает.

## Автоматические функции

Анти-спам работает всегда: если собеседник пишет больше 5 сообщений за 5 секунд, лишние сообщения удаляются.

Если включен режим "я занят", бот отвечает на первое сообщение, потом молчит 3 сообщения, потом снова может ответить.

## Деплой на Railway

1. Залей эту папку в GitHub-репозиторий (или используй `railway up` из CLI прямо из папки).
2. В Railway создай проект → Deploy from GitHub repo.
3. В Variables добавь `BOT_TOKEN` и `ADMIN_USER_ID` (значения из своего `.env`, сам файл `.env` в репозиторий не клади — он в `.gitignore`).
4. Railway сам подхватит `railway.json` / `Procfile` и запустит `python bot.py`.
5. База `helper_bot.sqlite3` живёт на диске контейнера и не подключена к постоянному Volume — при пересоздании сервиса или редеплое с нуля она обнулится, все пользователи и настройки удалятся. Это ожидаемо и ничего страшного, если это устраивает.

## Установка на VPS

```bash
sudo apt update
sudo apt install -y python3 python3-venv

cd /root/helpbot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
nano .env
```

Минимально в `.env` нужно:

```env
BOT_TOKEN=токен_из_BotFather
ADMIN_USER_ID=твой_числовой_id
```

Узнать `ADMIN_USER_ID` можно командой `/id` в личке с ботом.

## Подключение Telegram Business

1. В `@BotFather`: `/mybots` -> твой бот -> `Bot Settings` -> `Business Mode` или `Secretary Mode` -> `Turn On`.
2. В Telegram: `Настройки` -> `Telegram Business` -> `Chat Bots`.
3. Вставить username бота.
4. Дать права на чтение, ответы и удаление сообщений.
5. Выбрать чаты, где бот может работать.

## Автозапуск

```bash
sudo cp telegram-helper-bot.service /etc/systemd/system/helpbot.service
sudo systemctl daemon-reload
sudo systemctl enable --now helpbot
sudo journalctl -u helpbot -f
```

Если папка не `/root/helpbot`, поменяй `WorkingDirectory` и `ExecStart` в service-файле.

## Важно

- Не выдавай боту лишние права на Stars, подарки, профиль и username.
- Бот может удалить только то, что Telegram разрешает удалить.
- Старые сообщения или сообщения без нужных прав могут не удалиться.
- Если клиент не прошел регистрацию через код, его Business-подключение игнорируется.
